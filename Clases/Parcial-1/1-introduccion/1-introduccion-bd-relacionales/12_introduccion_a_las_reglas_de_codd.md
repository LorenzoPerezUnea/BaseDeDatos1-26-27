# 12. Introducción a las Reglas de Codd

Cuando el Modelo Relacional comenzó a popularizarse, muchas empresas afirmaban que sus productos eran "relacionales".

Sin embargo, no todos seguían realmente las ideas propuestas por Edgar F. Codd.

Para evitar confusiones, Codd definió una serie de principios que permitían evaluar hasta qué punto un sistema podía considerarse verdaderamente relacional.

Estas recomendaciones son conocidas como ​**Reglas de Codd**​.

### ¿Qué son las Reglas de Codd?

Las Reglas de Codd constituyen un conjunto de criterios que describen cómo debería comportarse un Sistema Gestor de Bases de Datos Relacional.

No fueron concebidas como una lista de requisitos obligatorios, sino como un modelo ideal.

En la práctica, pocos sistemas cumplen todas las reglas al cien por cien.

### ¿Por qué son importantes?

Las reglas ayudan a comprender qué características distinguen a un verdadero sistema relacional de uno que simplemente utiliza tablas.

Gracias a ellas podemos entender mejor conceptos que estudiaremos durante el curso, como:

* Independencia de los datos.
* Integridad.
* Consistencia.
* Manipulación mediante un lenguaje declarativo.

### Un adelanto

En una clase posterior no estudiaremos las trece reglas en profundidad, pero sí analizaremos las más importantes y veremos cómo siguen influyendo en los Sistemas Gestores de Bases de Datos actuales.

### Ideas clave

* Las Reglas de Codd sirven para evaluar un Sistema Gestor de Bases de Datos Relacional.
* Representan un modelo teórico de funcionamiento.
* La mayoría de productos comerciales implementan gran parte de estas reglas.
* Constituyen una referencia histórica y técnica para comprender el Modelo Relacional.
