# GitHub como material del curso

Hasta hace algunos años era habitual que los profesores distribuyeran apuntes en formato PDF o mediante plataformas educativas cerradas. Aunque estos métodos siguen siendo útiles, presentan algunas limitaciones: resulta difícil mantener varias versiones de un documento, compartir cambios o acceder rápidamente al historial de modificaciones.

En este curso utilizaremos **GitHub** como repositorio oficial de todo el material docente.

### ¿Qué es GitHub?

GitHub es una plataforma basada en el sistema de control de versiones Git que permite almacenar proyectos, compartir código y colaborar entre varias personas.

Aunque nació orientado al desarrollo de software, actualmente también se utiliza para publicar documentación técnica, libros, manuales y proyectos educativos.

Todo el material del curso estará organizado mediante carpetas y archivos Markdown.

### Organización del repositorio

Cada clase tendrá su propio directorio.

```text
Clase_01/
│── README.md
│── 01_...
│── 02_...

Clase_02/
│── README.md
│── 01_...
│── 02_...
```

El archivo `README.md` actuará como portada de cada clase y contendrá un índice con enlaces al resto de documentos.

Esta estructura facilitará localizar rápidamente cualquier tema del curso.

### ¿Por qué utilizar GitHub?

GitHub ofrece varias ventajas importantes.

* Todo el material estará disponible desde cualquier ordenador con acceso a Internet.
* Las correcciones podrán publicarse inmediatamente.
* El historial permitirá consultar versiones anteriores de los documentos.
* Los estudiantes podrán descargar el repositorio completo o únicamente las partes que necesiten.

Además, GitHub es una herramienta ampliamente utilizada en el ámbito profesional, por lo que su uso durante el curso constituye una competencia adicional.

### Git y GitHub

Es importante distinguir ambos conceptos.

**Git** es el sistema de control de versiones.

**GitHub** es una plataforma que utiliza Git para alojar proyectos y facilitar la colaboración entre usuarios.

Durante este curso utilizaremos GitHub como plataforma de distribución del material, mientras que Git aparecerá progresivamente cuando aprendamos a gestionar versiones de nuestros proyectos.

### Caso práctico

La base de datos de la empresa comercial evolucionará durante todo el semestre.

Cada nueva práctica añadirá tablas, restricciones, consultas o procedimientos.

Gracias a GitHub siempre podremos consultar el estado del proyecto en cualquier momento y recuperar versiones anteriores si fuera necesario.

### Ideas clave

* GitHub será la plataforma oficial del curso.
* Cada clase estará organizada mediante directorios y archivos Markdown.
* El archivo `README.md` servirá como índice de cada sesión.
* Git es el sistema de control de versiones; GitHub es la plataforma que lo utiliza.
* El repositorio permitirá mantener actualizado todo el material del semestre.
