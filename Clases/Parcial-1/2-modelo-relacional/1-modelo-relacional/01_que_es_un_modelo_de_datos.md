# ¿Qué es un modelo de datos?

Cuando observamos el mundo que nos rodea encontramos una enorme cantidad de información. Una empresa tiene clientes, productos, empleados, proveedores, facturas, pedidos y almacenes. Un hospital gestiona pacientes, médicos, tratamientos y citas. Una universidad administra estudiantes, asignaturas, profesores y matrículas.

Si intentáramos almacenar toda esa información sin seguir ninguna organización, pronto aparecerían el desorden, la duplicidad de datos y los errores.

Antes de construir una base de datos necesitamos responder una pregunta fundamental:

> **¿Cómo vamos a representar la realidad?**

La respuesta a esa pregunta es el ​**modelo de datos**​.

### La realidad es demasiado compleja

El mundo real contiene una cantidad prácticamente infinita de detalles.

Pensemos en un cliente de una empresa.

Podríamos almacenar:

* Nombre.
* Apellidos.
* Dirección.
* Teléfono.
* Correo electrónico.
* Fecha de nacimiento.
* Fotografía.
* Color favorito.
* Música preferida.
* Número de zapatos.

Sin embargo, no toda esa información resulta útil para una empresa comercial.

Una base de datos no pretende copiar exactamente la realidad.

Su objetivo consiste en construir una ​**representación simplificada**​, conservando únicamente los datos necesarios para resolver un problema.

### Una representación de la realidad

Un modelo de datos puede entenderse como un conjunto de reglas que nos indican:

* Qué información vamos a almacenar.
* Cómo se organiza esa información.
* Qué relaciones existen entre los datos.
* Qué restricciones deben cumplirse.

Gracias a estas reglas es posible construir bases de datos coherentes y fáciles de mantener.

### Un plano antes de construir

Podemos comparar un modelo de datos con el plano de un edificio.

Antes de comenzar una obra ningún arquitecto coloca ladrillos al azar.

Primero diseña planos donde aparecen las habitaciones, las puertas, las instalaciones eléctricas y la distribución del espacio.

Solo cuando el diseño está terminado comienza la construcción.

Con las bases de datos ocurre exactamente lo mismo.

El modelo de datos es el plano sobre el que posteriormente construiremos la base de datos.

### Un lenguaje común

Otra ventaja de los modelos de datos es que proporcionan un lenguaje compartido.

Cuando un analista habla de entidades, atributos o relaciones, todos los miembros del equipo entienden exactamente a qué se refiere.

Esto facilita enormemente la comunicación entre analistas, desarrolladores, administradores de bases de datos y usuarios.

### Caso práctico

A lo largo del semestre diseñaremos la base de datos de una empresa comercial.

Antes de crear ninguna tabla debemos decidir qué elementos forman parte del sistema.

Por ejemplo:

* Clientes.
* Productos.
* Empleados.
* Pedidos.
* Facturas.

El modelo de datos nos permitirá organizar toda esa información antes de comenzar a implementar la base de datos en MySQL.

### Ideas clave

* Un modelo de datos es una representación simplificada de la realidad.
* Su objetivo es organizar la información de forma coherente.
* Actúa como un plano antes de construir una base de datos.
* Define qué datos se almacenarán y cómo se relacionarán entre sí.
* Todo diseño de una base de datos comienza con un modelo de datos.
