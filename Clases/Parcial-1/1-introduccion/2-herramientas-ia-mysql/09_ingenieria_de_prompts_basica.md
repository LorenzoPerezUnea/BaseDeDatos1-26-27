# Ingeniería de Prompts básica

Cuando interactuamos con una Inteligencia Artificial, el mensaje que escribimos recibe el nombre de ​**prompt**​.

Aunque pueda parecer simplemente una pregunta, en realidad un prompt es una instrucción que guía el comportamiento del modelo.

Aprender a redactar buenos prompts permite obtener respuestas más útiles, precisas y adaptadas a nuestras necesidades.

### ¿Qué debe contener un buen prompt?

En la mayoría de los casos, un prompt eficaz responde a cuatro preguntas:

1. ¿Quién debe actuar la IA?
2. ¿Qué tarea debe realizar?
3. ¿Qué información de contexto necesita?
4. ¿Qué formato debe tener la respuesta?

Por ejemplo:

```text
Actúa como profesor universitario de Bases de
Datos.

Explica qué es una clave primaria utilizando un
lenguaje sencillo y dos ejemplos relacionados con
una empresa comercial.

Resume la explicación en menos de 300 palabras.
```

Este prompt proporciona mucha más información que una simple pregunta como:

```text
¿Qué es una clave primaria?
```

### Proporcionar contexto

El contexto ayuda a la IA a adaptar la respuesta.

Por ejemplo, no es lo mismo preguntar:

```text
Explícame SQL.
```

que indicar:

```text
Soy estudiante de primer curso de Ingeniería
Informática y estoy aprendiendo MySQL.
```

La segunda petición permite ajustar el nivel de dificultad.

### Especificar el formato

También es recomendable indicar cómo queremos recibir la información.

Podemos solicitar:

* Texto explicativo.
* Tabla comparativa.
* Lista de pasos.
* Código SQL.
* Ejercicios.
* Cuestionarios.
* Diagramas Mermaid.

Esto hace que la respuesta sea mucho más útil para el objetivo que perseguimos.

### Refinar la conversación

Una de las ventajas de los modelos conversacionales es que permiten mejorar progresivamente las respuestas.

Podemos realizar preguntas como:

* Explica el concepto con más detalle.
* Utiliza un ejemplo diferente.
* Simplifica la explicación.
* Añade un diagrama.
* Corrige el código anterior.

Cada nueva interacción aprovecha el contexto de la conversación anterior.

### Caso práctico

A lo largo del semestre utilizaremos prompts para pedir ayuda en tareas como:

* Revisar consultas SQL.
* Explicar errores de sintaxis.
* Generar datos de prueba.
* Diseñar pequeños modelos de bases de datos.
* Comparar distintas alternativas de diseño.

Con la práctica aprenderemos que un buen prompt suele ahorrar mucho tiempo y producir respuestas de mayor calidad.

### Ideas clave

* Un prompt es la instrucción que recibe la Inteligencia Artificial.
* Cuanto más claro sea el prompt, mejor será la respuesta.
* Es recomendable indicar el contexto y el formato deseado.
* La conversación puede refinarse mediante preguntas sucesivas.
* Saber formular buenos prompts es una competencia cada vez más importante para los profesionales de la informática.
